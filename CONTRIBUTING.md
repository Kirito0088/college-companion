# Contributing

## Purpose

This project follows a documentation-first workflow.

Every feature should be documented before implementation.

---

## Workflow

Documentation

↓

Approval

↓

Implementation

↓

Review

---

## Coding Standards

* Follow project documentation.
* Keep widgets small.
* Reuse components.
* Avoid duplicate code.
* Never hardcode colors or spacing.
* Respect the existing architecture.

---

## Before Opening a Pull Request

Verify:

* Code builds successfully.
* Documentation is updated (if required).
* No debug code remains.
* No unnecessary dependencies were added.

---

## Branch Naming

feature/attendance

feature/calendar

bugfix/login

refactor/dashboard

---

## Commit Messages

Examples:

feat: add attendance tracker

fix: resolve login issue

refactor: simplify assignment repository

docs: update screen specifications
